Digital Alchemy blueprints offer three distinct complexity paths for every app idea. Same app concept. Different technical implementation. You choose based on your current skill level and growth goals.

## How to Choose Your Path

<AccordionGroup>
  <Accordion title="I've never coded before" icon="seedling">
    **Start with Beginner.**
    
    Don't skip ahead. The Beginner path teaches you how to work with AI, read code, and deploy apps. These fundamentals matter more than technical complexity.
    
    Build 2-3 Beginner apps before moving to Intermediate.
  </Accordion>
  
  <Accordion title="I've built a few apps already" icon="tree">
    **Go Intermediate.**
    
    If you've completed at least 1 Beginner app and feel comfortable with HTML/CSS/JS basics, you're ready for Intermediate.
    
    Build 3-5 Intermediate apps before attempting Advanced.
  </Accordion>
  
  <Accordion title="I want user accounts and databases" icon="database">
    **Try Advanced, but complete the prerequisites first.**
    
    Advanced blueprints assume you understand:
    - Multi-file project structure
    - API requests and responses
    - localStorage and state management
    - Deployment workflows
    
    If any of those feel shaky, build more Intermediate apps first.
  </Accordion>
  
  <Accordion title="Can I skip levels?" icon="forward">
    **You can, but you shouldn't.**
    
    Each level builds on skills from the previous one. Skipping Beginner means you miss:
    - How to craft effective prompts
    - How to read and modify AI-generated code
    - How to debug when things break
    - How localStorage works
    
    These skills are prerequisites for everything that comes after.
  </Accordion>
</AccordionGroup>

## Progression Path

The recommended learning progression:

<Steps>
  <Step title="Week 1-2: First Beginner App">
    Pick any Beginner blueprint that interests you. Build it. Deploy it to GitHub Pages. Share the URL.
  </Step>
  
  <Step title="Week 3-4: Second Beginner App">
    Pick a different niche. Build faster this time. Notice what you understand now that you didn't before.
  </Step>
  
  <Step title="Week 5-6: Third Beginner App">
    By the third app, you should be able to modify AI-generated code confidently and add custom features.
  </Step>
  
  <Step title="Week 7-10: First Intermediate App">
    Upgrade one of your Beginner apps to Intermediate. Add API integration. Deploy to Vercel with a custom domain.
  </Step>
  
  <Step title="Week 11-16: More Intermediate Apps">
    Build 2-3 more Intermediate apps in different niches. Master multi-file architecture and API patterns.
  </Step>
  
  <Step title="Week 17+: First Advanced App">
    When you're ready, attempt your first full-stack app with Supabase. Expect it to take 3-4 weeks. That's normal.
  </Step>
</Steps>

<Note>
These timelines assume 5-10 hours of building per week. Adjust based on your schedule.
</Note>

## Key Differences Summary

| Aspect | Beginner | Intermediate | Advanced |
|--------|----------|--------------|----------|
| **Files** | 1 HTML file | 3-5 files | 15-25 files |
| **Storage** | localStorage | localStorage + APIs | PostgreSQL database |
| **Auth** | None | None | Email + OAuth |
| **Hosting** | GitHub Pages | Vercel | Vercel + Supabase |
| **Security** | Basic validation | Input validation | RLS + server actions |
| **Sync** | Single device | Single device | Multi-device real-time |
| **Deployment** | 5 minutes | 15 minutes | 30-60 minutes |
| **Maintenance** | None | Minimal | Regular updates |
